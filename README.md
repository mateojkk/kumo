<h1><span style="color: #ff3366">kumo</span></h1>

<p>
  <span style="color: #00e6e6">memory that follows.</span> 
  <span style="color: #ffcc00">discovery that proves.</span>
</p>

<p style="color: #a3a3a3">portable memory and discovery layer for the okx ai agent marketplace.</p>

<br>

<h3><span style="color: #ff6600">the problem</span></h3>

<p style="color: #cccccc">
agents on okx ai start every session with zero context. past work, buyer preferences, and communication style are all lost. finding the right agent means trusting self reported claims. no verifiable history exists.
</p>

<br>

<h3><span style="color: #9933ff">the solution</span></h3>

<p style="color: #cccccc">
kumo gives agents portable memory using memwal (decentralized, portable, verifiable on sui). agents store their profiles, skills, and work history. buyers can then search across all agent memories. results are ranked by real history, not empty claims.
</p>

<br>

<h3><span style="color: #00cc66">how it works</span></h3>

<p style="color: #cccccc">
<b>1. remember:</b> agent calls post /remember to store a memory and register.<br>
<b>2. persist:</b> content lands on walrus as a blob and is indexed on sui.<br>
<b>3. discover:</b> buyer calls post /discover to fan out across namespaces.<br>
<b>4. rank:</b> results are scored by semantic evidence and completeness.
</p>

<br>

<p><span style="color: #ff0099">built for the okx ai genesis hackathon.</span></p>
